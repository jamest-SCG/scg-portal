 When the project starts, the Project managers will be handed the project with the baseline budget for each applicable cost code for that particular project. Then as the job progresses, east cost code budget will need to be verified or adjusted by the PM on a monthly basis so we can reflect an accurate "cost to complete" for the project. One question that comes to mind, if we are importing these CSV files on a monthly basis to update the actual costs to date, what if a PM has previously revised an estimated cost to complete value. That value will remain the old value foundation (the CSV file), so we don't want the Project Managers updated cost to complete from the last month to be wiped out when importing the current months CSV file. We would just want the actual cost for each cost code to be updated at that point. So maybe there is an intial project set up where the original budget data is imported once but then never again?

These CSV reports also contain the original contract, the change orders and the revised contract value as well as the previously billed to date data. So that can be used to update the remaining billings that the PMs need to project on their monthly report
